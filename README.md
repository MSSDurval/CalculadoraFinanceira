# _Calculadora Financeira_

## 1. DIAGNÓSTICO E TEORIZAÇÃO

### 1. Identificação das partes envolvidas e parceiros
**Partes Envolvidas:**
- Contadores  
- Correntistas  
- Bancários  
- Usuários que desejam calcular juros de boletos  

### 2. Situação-problema identificada
A equipe do projeto, através de encontros e conversas, detectou que as partes envolvidas tinham dificuldades em calcular valores de seus boletos, impactando negativamente na contabilidade de usuários de cartões de crédito, de contadores e correntistas.

## 3. Demanda sociocomunitária e motivação acadêmica
A falta de métodos de calcular juros simples ou compostos gerava falhas graves na contabilidade de gastos e investimentos que eram realizados por diversos profissionais no ramo das finanças.  
Por isso, a equipe desenvolveu, usando tecnologias móveis, uma ferramenta virtual (software) capaz de realizar cálculos de juros que fosse acessível e disponível em dispositivos móveis, trazendo mobilidade aos profissionais da área.

### 4. Objetivos a serem alcançados
A equipe desenvolveu um software capaz de calcular juros simples e compostos.  
Foram utilizadas as seguintes tecnologias:
- React Native  
- Expo Framework  
- Visual Studio Code  
- Git  

---

## 2. PLANEJAMENTO PARA DESENVOLVIMENTO DO PROJETO

### 1. Plano de trabalho com cronograma das atividades
O projeto foi planejado e discutido pela equipe responsável, que estabeleceu um cronograma de objetivos a serem realizados. Os objetivos definidos foram:
- Preparar o ambiente, configurando e instalando programas e recursos necessários  
- Escolher a linguagem e framework  
- Definir metas diárias para a construção dos componentes e módulos  
- Criar a documentação do projeto  
- Testar o software  
- Entregar o produto final às partes interessadas  

### 2. Metodologia
O projeto foi criado utilizando modelos e protótipos de outros projetos.  
Para a Calculadora de Juros, foi decidido dividir cada módulo como parte da calculadora — cada arquivo representando uma parte específica do software.

### 3. Avaliação dos resultados alcançados
A eficácia do projeto será avaliada através de questionários e feedbacks com profissionais do ramo de finanças e partes interessadas.  
Também será comparada a facilidade de uso e a curva de aprendizado dos usuários.
